## 1. 한 줄 정의

가족 4명(효성·소연·효준·미희)의 싱가폴 여행을 **계획·기록·정산**하는 1페이지 웹앱. 노션 DB 연동, Firestore 실시간 동기화, 별점·코멘트 공유.

## 4. 화면 구조

**앱 스타일 화면 분리** — 단일 HTML 안에서 view 단위로 진짜 전환 (무드보드 톤).
**하단 네비게이션 바** 4개:

1. **홈** — 여행 개요, D-day 카운트다운, 멤버 4명, 주요 정보(숙소·항공편)
2. **일정** — 타임라인 + 지도 (제주의 메인)
3. **코멘트** — 활동별 코멘트 모음 + 일자별 총평
4. **비용** — 카테고리별 지출, 정산

각 view는 화면 단위로 전환 (스크롤 X, 탭 시 슥 바뀜).

## 5. 카드 디자인 (핵심)

### 사진 없는 카드 — 아이콘 + 컬러바

```
┌─────────────────────────────────────────────┐
│ ▌  [✈️]   창이공항 도착            14:30   │
│ ▌         🛬 Singapore Airlines SQ607        │
│ ▌         💬 코멘트 2개                       │
│ ▌         ─────────────────────              │
│ ▌         효성 ●4   소연 ●5                   │
└─────────────────────────────────────────────┘
   ↑좌측 컬러바 (카테고리 색)
```

### 카테고리 × 아이콘 × 컬러

| 카테고리 | 아이콘 | 컬러바 |
|---|---|---|
| 비행 | ✈️ | 민트 |
| 호텔 | 🏨 | 라벤더 |
| 관광 | 🎡 | 핑크 |
| 식사 | 🍽️ | 옐로우 |
| 이동 | 🚇 | 그레이 |
| 쇼핑 | 🛍️ | 코랄 |
| 카페·휴식 | ☕ | 베이지 |

> 아이콘은 이모지 vs SVG(Lucide) — 통일감 위해 **SVG 라인 아이콘** 권장. 이모지는 OS마다 모양 다름.

## 6. 별점 시스템

- **1~5 숫자 버튼** 노출, 탭해서 선택 (제주의 별 클릭 패턴과 동일)
- **0점(무평가)** 지원: 같은 숫자 다시 탭하면 해제 (제주와 동일)
- **카드 메인**에는 **매긴 사람만** 표시: `효성 ●4   소연 ●5`
  - ● 색깔 = 사람 고정색
  - 매기지 않은 사람은 표시 안 함
- 본인 별점 입력 UI는 카드 펼침 시 노출 (또는 코멘트 패널 내)

---

## 7. 코멘트

- 활동별 코멘트 (제주 패턴 유지)
- 메인 카드엔 **💬 N개** 카운트만
- 탭 시 펼침 — 작성자 이름(고정색) + 텍스트 + 시간
- 4명 모두 작성 가능

---

## 8. 비용

- **기본 통화: SGD** — 숫자만 입력 (예: `28.50`)
- **한국 결제 분**: 원화 단위 입력 (예: `25000원`) — 정산 시 환산
- **환율**: 정산 화면에서 **수동 입력**(예: 1 SGD = 1,000원) → 자동 합산
- **카테고리별 합계** 표시 (비행/호텔/식사/관광/쇼핑/기타)
- **누가 결제** 필드 (효성/소연/효준/미희 중 선택)
- **정산 뷰**: 1인당 부담액 vs 실제 지출 → 누가 누구한테 얼마

## 9. 노션 연동 — 클립보드 복사 방식 (제주와 동일)

API 연동 ❌. **"클립보드에 복사 (노션 붙여넣기용)"** 버튼 하나로 끝.

```
- 활동명 [카테고리 · 28.50 SGD · 효성:4/5 · 소연:5/5 · ...]
  - 효성: 코멘트 텍스트
  - 소연: 코멘트 텍스트
```

- 별점 매긴 사람만 노출
- SGD 그대로 표기 (KRW 결제는 `25,000원` 형식)
- 일자별 총평 포함
